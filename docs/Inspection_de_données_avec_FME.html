Inspection de données avec FME

Objectif

Première prise en main du logiciel FME, nous allons simplement inspecter (visualiser) une donnée géographique, sans y appliquer de modifications.

Ressource

Workspace :

  • \ex_ visualisation\ ex_visualisation_start.fmw , nous débutons l’exercice avec un workpace vide.

Données :

  • \ex_ visualisation\ ressources\communes_VD_GE_NE_FR_VS_BE.shp – fichier Shapefile des cantons romands. 

Exercice

Démarrer FME Workbench

Cliquer sur FME Desktop > FME Workbench dans le menu démarrer.

Ajout du MNT(reader)

Cliquer sur l’onglet « Main ».

Puis sur :

Puis ajouter le type de format shape file comme suit :

Le Reader (input) doit être défini comme suit :

Le fichier ESRI Shape file "communes_VD_GE_NE_FR_VS_BE.shp" est sélectionné.

Puis valider en cliquant sur « ok ».

Un nouvel élément est apparu dans la fenêtre principale. Il s’agit du reader que l’on vient d’ajouter.

Ajout d’un inspector

L’inspector est l’outil (le transformer) qui nous permettra de visualiser la donnée lorsque la transformation sera exécutée.

Pour ce faire, cliquer droite dans la fenêtre principale, puis sélectionner « Insert inspector ».

Cela place l’Inspector dans la fenêtre principale.

L’étape suivante consiste à lier le reader à notre outil d’inspection (inspector), pour ce faire nous allons créer une liaison entre ces deux éléments. Cela s’effectue en cliquant sur le triangle blanc à droite du reader (output port) puis sur le triangle rouge à gauche de l’inspector (input port).

Cela crée graphiquement une liaison entre ces deux éléments. Ainsi les données qui seront lues à travers le reader vont directement être transmises dans l’inspector  pour y être visualisées.

Exécuter le traitement

Puis en exécutant le workspace dans la barre des outils ou simplement en cliquant sur :

La lecture s’effectue, nous constatons que 1146 objets ont été lus depuis ce fichier ESRI Shape File.

Visualiser la donnée

Une fenêtre en bas de l’écran s’est ouverte. Il s’agit de l’inspector.

Vous pouvez sans autre zoomer, dé-zoomer, interroger un objet en cliquant sur la carte, visualiser ces attributs.

Conclusion

L’inspector est un outil que l’on place tout au long de la transformation et qui nous permet de visualiser l’état de la donnée à des endroits différents. Lors du développement de grands workspaces (avec peut-être plus de 1000 transformer), le développeur place des inspectors aux endroits stratégiques pour s’assurer que la transformation s’effectue comme souhaité.
