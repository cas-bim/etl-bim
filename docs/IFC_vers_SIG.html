Objectif

Afin d'illustrer la transformation d'un modèle BIM en format SIG, nous allons utiliser la pateforme Cesium ion.

Cesium ion est une plateforme robuste, évolutive et sécurisée pour les données géospatiales 3D. Téléchargez votre contenu et Cesium ion l'optimisera sous forme de tuiles 3D, l'hébergera dans le nuage et le diffusera sur n'importe quel appareil.

Cesium ion comprend l'accès à des contenus 3D mondiaux et sélectionnés, notamment Cesium World Terrain, l'imagerie Bing Maps et Cesium OSM Buildings. Combinez ces ressources avec vos propres données pour les voir dans un contexte géospatial, et partagez les résultats en toute sécurité avec vos collègues ou mettez-les à disposition via vos applications.

Plus d'info sur : https://cesium.com/platform/cesium-ion/

Ajout des arbres

Le standard IFC 4.x ne possède pas de classes pour les arbres, cette information se trouve dans la classe générique "IfcGeographicElement", parmis plein d'autres éléments (les bien-fonds, les données RDPPF, etc..). Il faudra donc filtrer les éléments de cette classe, à l'aide du seul paramètre que les distingue ; les properties set.

Pour ce faire, nous allons connecter la sortie du FeatureReader "IfcGeographicElement" à un "GeometryPropertyExtractor"


Ce transformer va extraire les properties sets en attributs. Par contre ces attributs sont dynamiques et donc non visibles dans FME. Cela est dû au fait que la structure des properties set n'est pas connue et peut varier d'un objet à l'autre. C'est une notion de FME avancé, que l'on ne va pas aborder dans ce cours.

Pour faire apparaître l'attribut que nous permet de distinguer les arbres des autres éléments, nous allons "exposer" l'attribut "Classe" à l'aide d'un "AttributeExposer".

Cet attribut nommé "CLASSE" est celui qui contient la classe d'arbre. Ainsi, si cet attribut est complété, cela signifie que l'objet est un arbre. Si cet attribut est vide, cela n'est pas un arbre.
